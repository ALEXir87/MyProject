1.Проиницилизировать репозиторий: 
> git init - инициализация локального репозитория
2.Создать репозиторий:
> git add filename - добавляется файл с именем filename
> git add . - добавляются  все файлы
3.Прокомментировать (сохранить изменения с комментарием):
> git commit -m "текст комментария" - добавление информации о вашем действии 4.Журнал commit:
> git log- журнал изменений (commit)
5.Перемещаться по commit:
> git checkout код commit - переход на выбранный commit
> git checkout master - переход к последнему commit
6.Изменения с commit версией файла:
> git diff - разницу между текущим файлом и закоммиченным(commit) файлом
7.Вывести список всех веток на экран:
> git branch
8.Создать новую ветку:
> git branch branch_name - создается ветка с именем branch_name
9.Перейти к уже созданной,нужной нам ветке branch_name:
> git checkout branch_name - переход к нужной ветке branch_name
10.Удалить ненужную ветку branch_name:
> git branch -d branch_name - удаление ветки branch_name
11.Отобразить список созданных веток:
> git log --graph - отображение созданных веток
12.Сделать слияние веток:
> git merge branch_name - слияние ветки branch_name
13.Сделать копию репозитория из удалённого сервиса GitHub.com в локальный репозиторий:
> git clone адресс удалённого репозитория
14.Локальный репозиторий отправить на удалённый сервис GitHub.com: 
> git push
15.Синхронизировть изменения на удалённом репозитории GitHub.com с локальным репозиторием:
> git pull
